# Эксперименты

## Обзор

Раздел **Эксперименты** — это библиотека сущностей и запуск матричных оценок:

1. **Алгоритмы** — именованные конфигурации поиска (режим, веса, фильтры, размер пула кандидатов KNN).
2. **Шаблоны запросов** — текст запроса, поле **`relevant_ids`**, и **`index`** (коллекция по умолчанию, часто **`all`**).
3. **Запуски бенчмарка** — вы выбираете набор алгоритмов и шаблонов; платформа выполняет **асинхронную матрицу N×M** и сохраняет в **MongoDB** метрики: **NDCG@K**, **MRR**, **Precision@K**, **Recall@K**, **latency_ms**, **score_separation** и др.

Всё выполняется в контексте **текущего активного профиля** из **Настроек**.

## Алгоритмы

Алгоритм описывает **как** искать, а не **где**:

| Поле | Смысл |
|------|--------|
| `name`, `description` | Подписи для людей |
| `mode` | `bm25`, `semantic`, `hybrid`, `rrf` |
| `bm25_weight`, `knn_weight` | Для **hybrid** сумма весов должна быть **1.0** |
| `num_candidates` | Размер пула для KNN (ограничения задаёт API) |
| `filters` | Необязательные `filter_term`, `filter_gte`, `filter_lte` |

**Важно:** у **Algorithm нет поля `index`**. Индекс для конкретного прогона задаётся в **шаблоне запроса** (поле `index`, по умолчанию **`all`**). Один и тот же алгоритм можно комбинировать с шаблонами на разные коллекции.

## Шаблоны запросов

| Поле | Смысл |
|------|--------|
| `name` | Короткое имя |
| `query` | Текст запроса к OpenSearch |
| `index` | Логический ключ индекса (по умолчанию **`all`**) |
| `relevant_ids` | ID документов, считающихся релевантными (можно дополнять позже через PUT) |
| `notes` | Произвольные заметки |

В UI выбор индекса может выглядеть как **мультивыбор**, но в модели хранится одно значение ключа — смотрите фактическое поведение интерфейса.

## Запуск бенчмарка

Параметры:

- **`algorithm_ids`** — какие сохранённые алгоритмы участвуют
- **`template_ids`** — какие шаблоны участвуют
- **`k`** — отсечка для метрик **@K**
- **`size`** — сколько результатов запрашивать (**не меньше `k`**)

**Структура результата:**

- **`results[algorithm_id][template_id]`** — метрики и детали по паре
- **`summary`** — агрегаты **по алгоритму** (усреднение по шаблонам)

Подробнее о метриках — в [Метрики](metrics.md).

## Интерфейс

Во вкладке **Эксперименты**:

- **Алгоритмы** — библиотека конфигураций
- **Шаблоны** — запросы и разметка
- **Бенчмарк** — настройка матрицы и просмотр результатов

Активный **профиль подключения** определяет кластер, эмбеддинги и доступные логические ключи индексов.
