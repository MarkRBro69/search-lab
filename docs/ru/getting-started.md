# Быстрый старт

## Требования

Нужны:

- **Python 3.14+** и **[uv](https://docs.astral.sh/uv/)** — зависимости и запуск приложения
- **Docker** с Compose — локальные **OpenSearch** и **MongoDB**

Полный чеклист (`.env`, pre-commit и т.д.) — в [настройке окружения разработчика](../developer-setup.md). Здесь только кратко.

## Запуск стека

Из корня репозитория:

```powershell
uv sync --all-extras

docker-compose up -d opensearch mongodb

# По желанию: тестовые документы
uv run python scripts/seed.py

uv run uvicorn src.main:app --reload
```

Если в проекте есть цель `make dev`, она обычно запускает то же с hot-reload.

По умолчанию:

- **Веб-интерфейс:** `http://localhost:8000`
- **Swagger:** `http://localhost:8000/docs`

## Интерфейс

В шапке три основные **вкладки**:

| Вкладка | Назначение |
|---------|------------|
| **Поиск** | Запросы, explain, разметка для eval, rank-eval |
| **Эксперименты** | Алгоритмы, шаблоны, бенчмарки |
| **Настройки** | Профили подключения: создание, правка, активация, проверка |

В **шапке** отображается **активный профиль подключения** после активации в «Настройках». Поиск и операции с документами используют **его** OpenSearch, эмбеддинги и карту логических индексов.

### Вкладка «Поиск»

В боковой панели обычно есть:

- **Режим** — `bm25`, `semantic`, `hybrid` или `rrf`
- **Коллекция** — логический ключ индекса (в т.ч. `all`, где поддерживается)
- **Размер** — сколько документов вернуть
- **Explain** — разбор оценки (гибрид/RRF) или дерево BM25, где применимо
- **Фильтры** — необязательные ограничения

Дополнительно:

- **Eval marks** — отметка релевантных ID и оценка качества в духе **POST /api/v1/eval**
- **Rank-eval** — пакетная BM25-оценка через **POST /api/v1/eval/rank-eval** (один индекс, не `all`)

### Вкладка «Эксперименты»

Подвкладки:

- **Алгоритмы** — сохранённые конфигурации поиска
- **Шаблоны** — запросы и списки релевантных ID
- **Бенчмарк** — выбор N×M, запуск, просмотр результатов

Бенчмарк использует **тот же активный профиль**, что и поиск.

### Вкладка «Настройки»

Управление **профилями**: создать, изменить, **активировать**, **протестировать** связь с OpenSearch и эмбеддингами.

## Первый сценарий

1. Откройте **Настройки**, **создайте** профиль (или выберите существующий) с корректными OpenSearch и эмбеддингами.
2. **Активируйте** нужный профиль.
3. По желанию нажмите **Тест** и убедитесь, что подсистемы отвечают.
4. Перейдите в **Поиск**, задайте **режим**, **коллекцию**, **размер**, введите **запрос** и выполните поиск.

Дальше можно размечать релевантность или перейти к **Экспериментам** для библиотечных бенчмарков.

## Режимы поиска

| Режим | Суть | Заметки |
|-------|------|---------|
| **bm25** | Ключевой поиск (TF-IDF/BM25, для отображения нормализуется) | Удобен для точных формулировок и идентификаторов |
| **semantic** | KNN по векторам | Удобен для перефразировок и описательных запросов |
| **hybrid** | Взвешенная смесь BM25 и KNN | Веса **`bm25_weight`** и **`knn_weight`** в сумме дают **1.0** |
| **rrf** | Reciprocal Rank Fusion двух списков | Слияние по рангам без ручного подбора весов как в гибриде |

### Фильтры

Необязательные фильтры совпадают по смыслу с API:

- **`filter_term`**
- **`filter_gte`** — строки вида `"поле:значение"`
- **`filter_lte`** — аналогично

При поиске по нескольким физическим индексам неизвестные поля могут игнорироваться — интерфейс остаётся доменно-нейтральным.

## Важные HTTP-эндпоинты

Маршруты API ниже с префиксом **`/api/v1`**, кроме health и документации:

| Метод | Путь | Назначение |
|-------|------|------------|
| GET | `/api/v1/search` | Поиск |
| GET | `/api/v1/search/explain/{index_key}/{doc_id}` | Объяснение BM25 для документа (параметр `q`) |
| POST | `/api/v1/eval` | Один запрос + релевантные ID → NDCG@K, MRR, Precision@K, Recall@K |
| POST | `/api/v1/eval/rank-eval` | Пакетный BM25 `_rank_eval` (один индекс, не `all`) |
| POST | `/api/v1/documents/{index_key}` | Создать документ |
| GET | `/api/v1/documents/{index_key}/{doc_id}` | Получить документ |
| PUT | `/api/v1/documents/{index_key}/{doc_id}` | Обновить документ |
| DELETE | `/api/v1/documents/{index_key}/{doc_id}` | Удалить документ |
| GET/POST/PUT/DELETE | `/api/v1/profiles` … | Профили (см. [profiles.md](profiles.md)) |
| POST | `/api/v1/profiles/{id}/activate` | Активация профиля |
| POST | `/api/v1/profiles/{id}/test` | Проверка подключений |
| GET/POST/DELETE | `/api/v1/experiments/algorithms` … | Алгоритмы (см. [experiments.md](experiments.md)) |
| GET/POST/PUT/DELETE | `/api/v1/experiments/templates` … | Шаблоны |
| POST/GET/DELETE | `/api/v1/experiments/benchmark` … | Бенчмарки |

**Вне `/api/v1`:**

| Метод | Путь |
|-------|------|
| GET | `/health` |
| GET | `/docs` |
