# Профили подключения

## Что такое профиль

**Профиль подключения** объединяет настройки, без которых Search Lab не работает с вашими данными:

- **OpenSearch** — хост, порт, TLS, способ аутентификации
- **Эмбеддинги** — как строятся векторы для запросов и документов (локальная модель или облако)
- **Логические индексы** — соответствие **ваших ключей** (например, `articles`) **именам индексов** в кластере
- **Поля BM25** — для каждого логического ключа список текстовых полей для ключевого поиска

Можно хранить **несколько** профилей и **активировать** один. Поиск, CRUD документов, eval и бенчмарки используют **активный** профиль.

## Логические индексы и `all`

- Каждый **логический ключ** указывает на один индекс OpenSearch (имя задаёте вы).
- Зарезервированный ключ **`all`** означает **поиск сразу по всем** настроенным в профиле индексам.

**Запись** (создание / обновление / удаление документа) для ключа **`all` запрещена** — нужно указать **конкретный** логический ключ, чтобы сервис знал целевой индекс.

## Настройки OpenSearch

| Аспект | Что задаёте |
|--------|-------------|
| Хост и порт | Куда обращаться по HTTP(S) |
| `use_ssl` | Использовать ли HTTPS |
| `auth_type` | **`none`**, **`basic`** (логин и пароль) или **`aws_signature_v4`** (подпись SigV4; регион и ключи — по требованиям AWS) |

Точные имена полей смотрите в Swagger при создании или замене профиля.

## Бэкенды эмбеддингов

| Провайдер | Типичное применение |
|-----------|---------------------|
| **`local_sentence_transformers`** | Локальная модель по **имени** (часто задаётся окружением) — удобно для офлайн-разработки |
| **`aws_bedrock`** | Модель в **Bedrock**; могут понадобиться **учётные данные AWS** и регион в профиле |

## Политика секретов

**Пароли, ключи API и секреты AWS не возвращаются** ни в списке профилей, ни при чтении одного профиля. В ответе остаются только **безопасные** поля (хост, порт, **тип** auth, при необходимости имя пользователя **без** пароля).

При **редактировании** профиля секреты нужно **передать заново** в теле запроса — сервер не присылает старые секреты обратно.

## Справочник API

Префикс **`/api/v1`**.

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/api/v1/profiles` | Список профилей (без секретов) |
| POST | `/api/v1/profiles` | Создать профиль |
| GET | `/api/v1/profiles/{profile_id}` | Один профиль |
| PUT | `/api/v1/profiles/{profile_id}` | Полная замена |
| DELETE | `/api/v1/profiles/{profile_id}` | Удалить |
| POST | `/api/v1/profiles/{profile_id}/activate` | Активировать |
| POST | `/api/v1/profiles/{profile_id}/test` | Проверить OpenSearch и эмбеддинги |

## Интерфейс (вкладка «Настройки»)

Здесь вы:

- **Создаёте** и **редактируете** профили
- **Активируете** нужный профиль для поиска и экспериментов
- **Тестируете** соединение и видите успех/ошибку по подсистемам

В **шапке** отображается **активный профиль**, чтобы было ясно, какой кластер и карта индексов сейчас используются.
